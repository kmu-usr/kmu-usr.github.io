## 第一部分：註冊 GitHub 帳號（約 10 分鐘）

1. 開啟 **https://github.com** → 點右上角 **Sign up**

2. 依序輸入：
   - **Email**：計畫公用信箱
   - **Password**：設一組強密碼（**立刻記入計畫交接文件**）
   - **Username**：輸入 `kmu-usr`
     - 輸入時頁面會即時檢查：綠色勾＝可用，直接用
     - 若顯示已被佔用，依序改試：`kmu-usr-tw` → `kmuusr` → `kmu-usr-official`
     - ⚠ **帳號名決定網址**。若改用備案，後續所有步驟的 `kmu-usr` 都替換成實際帳號名，儲存庫名也要跟著改（見第二部分注意事項）

3. 通過人機驗證（拼圖或圖片題）→ 按 **Create account**

4. 到信箱收驗證碼（8 位數）→ 填回頁面

5. 後續引導問題（用途調查）可一路選 **Skip** 或隨意作答，不影響功能

6. **強烈建議**：設定雙重驗證
   - 右上角頭像 → Settings → 左側 **Password and authentication** → **Two-factor authentication** → Enable
   - 選 SMS（簡訊）最簡單；**備援碼（Recovery codes）列印後夾入交接文件**

---

## 第二部分：建立儲存庫（約 3 分鐘）

1. 登入後點右上角 **「＋」** → **New repository**

2. 填寫：
   - **Repository name**：輸入 `kmu-usr.github.io`
     - ⚠ **必須與帳號名完全一致**再接 `.github.io`——這是 GitHub 的特殊規則，符合此格式的儲存庫會直接掛在根網址（無路徑）
     - 若第一部分用了備案帳號名（如 `kmuusr`），這裡就要填 `kmuusr.github.io`
   - **Public / Private**：選 **Public**（Pages 免費版必須公開）
   - **Add a README file**：✅ 勾選

3. 按 **Create repository**

---

## 第三部分：上傳網站檔案（約 5 分鐘）

1. 進入剛建立的儲存庫頁面 → 點 **Add file** → **Upload files**

2. 開啟電腦上的本資料夾，**同時選取**：
   - `index.html`
   - 整個 `img` 資料夾
   - 一起拖曳到網頁的上傳區
   - ⚠ 確認上傳清單中圖片路徑顯示為 `img/xxx.jpg`（資料夾結構有帶到）

3. 下方 Commit changes 欄位保持預設 → 按 **Commit changes**

4. 回到儲存庫首頁，確認看得到 `index.html` 與 `img/`

---

## 第四部分：開啟 Pages（約 2 分鐘＋等待）

1. 儲存庫頁面 → **Settings**（右上齒輪）→ 左側選單 **Pages**

2. **Build and deployment** 區：
   - Source：**Deploy from a branch**
   - Branch：**main**　資料夾：**/ (root)** → 按 **Save**

3. 等 **1–3 分鐘**，重新整理 Pages 頁面，上方會出現：
   > Your site is live at `https://kmu-usr.github.io`

4. 點開網址驗收：
   - [ ] 首屏校徽、大標、照片正常
   - [ ] 往下滑：數字有跳動動畫
   - [ ] **用手機也開一次**（主戰場是手機）
   - [ ] 點「來信洽詢合作」按鈕，確認開啟郵件並帶入收件人

**完成。網址 `https://kmu-usr.github.io` 即可印上文宣、做 QR code。**

## 日常維護

### 更新內容（改字、換數字）
儲存庫 → 點 `index.html` → 右上鉛筆圖示 ✏ 線上編輯 → 改完按 **Commit changes** → 約 1 分鐘後自動生效。
（或由計畫窗口把需求交給 AI 改好新版 `index.html`，用 Upload files 覆蓋上傳。）

### 換照片
Upload files 上傳同檔名新圖即覆蓋（例如換 `img/hero.jpg`）。

### 啟用「下載完整提案書」按鈕（目前隱藏）
1. 準備可公開版 PDF（**須為去除手機號碼之修訂版**），檔名命名為 `proposal.pdf`
2. Upload files 上傳 `proposal.pdf` 至儲存庫根目錄
3. 編輯 `index.html`，找到這段：
   `<!-- 預留：完整提案書下載`
   刪除包住按鈕的 `<!--` 與 `-->` 註解符號 → Commit

### 改壞了想復原
儲存庫 → 上方 **Commits**（時鐘圖示）→ 找到想回去的版本 → 瀏覽該版檔案複製回來。每次 Commit 都有完整紀錄，不會真的弄丟。
